## Backend Deployment (Render.com)

### Step 1: Prepare Backend for Deployment

#### 1.1 Update requirements.txt
Make sure `tariff_auditor_backend/requirements.txt` includes:
```txt
fastapi>=0.111.0
uvicorn[standard]>=0.29.0
crewai>=0.51.0
crewai-tools>=0.8.3
pydantic>=2.7.0
pydantic-settings>=2.2.0
sqlalchemy[asyncio]>=2.0.29
aiosqlite>=0.20.0
python-dotenv>=1.0.1
httpx>=0.27.0
beautifulsoup4>=4.12.0
langchain-google-genai>=1.0.0
redis>=5.0.0
psycopg2-binary>=2.9.9
```

#### 1.2 Create render.yaml (Already created - see below)

#### 1.3 Update database.py for PostgreSQL
The system will automatically use PostgreSQL on Render via DATABASE_URL environment variable.

### Step 2: Deploy to Render.com

#### 2.1 Create New Web Service
1. Go to https://dashboard.render.com
2. Click **"New +"** → **"Web Service"**
3. Connect your GitHub repository
4. Select your repository: `TariffGuard-Ai`

#### 2.2 Configure Web Service
Fill in the following:

**Basic Settings:**
- **Name**: `tariffguard-backend` (or your preferred name)
- **Region**: Choose closest to your users (e.g., Oregon, Frankfurt, Singapore)
- **Branch**: `main` (or your default branch)
- **Root Directory**: `tariff_auditor_backend`
- **Runtime**: `Python 3`
- **Build Command**: `pip install -r requirements.txt`
- **Start Command**: `uvicorn main:app --host 0.0.0.0 --port $PORT`

**Instance Type:**
- Select **"Free"** (⚠️ Will spin down after 15 minutes of inactivity)

#### 2.3 Add Environment Variables
Click **"Advanced"** → **"Add Environment Variable"**

Add these variables:

| Key | Value | Notes |
|-----|-------|-------|
| `GEMINI_API_KEY` | `your_gemini_api_key` | Get from Google AI Studio |
| `FRONTEND_URL` | `https://your-app.vercel.app` | Update after deploying frontend |
| `DATABASE_URL` | (Auto-generated by Render) | Don't add manually |
| `SERPER_API_KEY` | (optional) | Leave empty if not using |
| `WCO_API_KEY` | (optional) | Leave empty if not using |
| `REDIS_URL` | (optional) | Leave empty if not using |

⚠️ **Important**: Don't set `DATABASE_URL` manually - Render will auto-generate it when you add PostgreSQL database.

#### 2.4 Add PostgreSQL Database
1. In your web service dashboard, click **"Environment"** tab
2. Scroll down to **"Add Database"**
3. Click **"Add Database"** → **"PostgreSQL"**
4. Select **"Free"** tier (90 days free, then $7/month)
5. Click **"Create Database"**

Render will automatically:
- Create PostgreSQL database
- Set `DATABASE_URL` environment variable
- Connect it to your web service

#### 2.5 Deploy
1. Click **"Create Web Service"**
2. Wait for deployment (5-10 minutes)
3. Check logs for any errors
4. Once deployed, you'll get a URL like: `https://tariffguard-backend.onrender.com`

### Step 3: Verify Backend Deployment

Test your backend:
```bash
# Health check
curl https://tariffguard-backend.onrender.com/health

# Should return: {"status": "healthy"}
```

⚠️ **First Request**: May take 30-60 seconds if service was sleeping (cold start).

## Frontend Deployment (Vercel)

### Step 1: Prepare Frontend

#### 1.1 Update API URL
Create `tariff_auditor_frontend/.env.production`:
```env
NEXT_PUBLIC_API_URL=https://tariffguard-backend.onrender.com
```

#### 1.2 Commit Changes
```bash
git add .
git commit -m "Add production environment variables"
git push origin main
```

### Step 2: Deploy to Vercel

#### 2.1 Import Project
1. Go to https://vercel.com/dashboard
2. Click **"Add New..."** → **"Project"**
3. Import your GitHub repository: `TariffGuard-Ai`
4. Click **"Import"**

#### 2.2 Configure Project
**Framework Preset**: Next.js (auto-detected)

**Root Directory**: 
- Click **"Edit"**
- Select `tariff_auditor_frontend`
- Click **"Continue"**

**Build Settings** (auto-detected):
- Build Command: `npm run build`
- Output Directory: `.next`
- Install Command: `npm install`

#### 2.3 Add Environment Variables
Click **"Environment Variables"**

Add:
| Key | Value |
|-----|-------|
| `NEXT_PUBLIC_API_URL` | `https://tariffguard-backend.onrender.com` |

⚠️ Replace with your actual Render backend URL!

#### 2.4 Deploy
1. Click **"Deploy"**
2. Wait 2-5 minutes
3. You'll get a URL like: `https://tariffguard-ai.vercel.app`

### Step 3: Update Backend CORS

Go back to Render.com and update `FRONTEND_URL`:
1. Open your backend service
2. Go to **"Environment"** tab
3. Update `FRONTEND_URL` to your Vercel URL: `https://tariffguard-ai.vercel.app`
4. Click **"Save Changes"**
5. Service will automatically redeploy

## Troubleshooting

### Backend Issues

#### ❌ "Service Unavailable" or 503 Error
**Cause**: Service is sleeping (cold start on free tier)
**Solution**: Wait 30-60 seconds and retry. First request wakes up the service.

#### ❌ "Database connection failed"
**Cause**: DATABASE_URL not set or incorrect
**Solution**: 
1. Check if PostgreSQL database is added in Render
2. Verify DATABASE_URL is auto-generated
3. Check database connection in logs

#### ❌ "GEMINI_API_KEY not found"
**Cause**: Environment variable not set
**Solution**: 
1. Go to Render dashboard → Environment
2. Add GEMINI_API_KEY
3. Save and redeploy

#### ❌ Rate Limit Errors
**Cause**: Gemini API free tier limit (15 requests/minute)
**Solution**: 
1. Wait 60 seconds between audits
2. Upgrade to paid Gemini API tier
3. Implement request queuing

### Frontend Issues

#### ❌ "Failed to fetch" or CORS Error
**Cause**: Backend FRONTEND_URL not matching Vercel URL
**Solution**:
1. Update FRONTEND_URL in Render to exact Vercel URL
2. Include https:// and no trailing slash
3. Redeploy backend

#### ❌ "API endpoint not found"
**Cause**: NEXT_PUBLIC_API_URL incorrect
**Solution**:
1. Check environment variable in Vercel
2. Ensure it points to Render backend URL
3. Redeploy frontend

#### ❌ Build Failed
**Cause**: Missing dependencies or TypeScript errors
**Solution**:
1. Check build logs in Vercel
2. Fix TypeScript errors locally
3. Test build locally: `npm run build`
4. Push fixes and redeploy

### Database Issues

#### ❌ "Enum value not found"
**Cause**: Old SQLite data incompatible with PostgreSQL
**Solution**:
1. Database will be fresh on Render (no migration needed)
2. If issues persist, check enum definitions in models

## Important Notes

### ⚠️ Free Tier Limitations

#### Render.com Free Tier:
- ✅ **Pros**: 
  - 750 hours/month free compute
  - PostgreSQL database (90 days free)
  - Automatic HTTPS
  - Custom domains
  
- ⚠️ **Cons**:
  - **Spins down after 15 minutes** of inactivity
  - **Cold start**: 30-60 seconds for first request
  - 512 MB RAM
  - Shared CPU

#### Vercel Free Tier:
- ✅ **Pros**:
  - Unlimited deployments
  - 100 GB bandwidth/month
  - Automatic HTTPS
  - Global CDN
  - No cold starts
  
- ⚠️ **Cons**:
  - 100 GB bandwidth limit
  - 6,000 build minutes/month

### 💡 Performance Tips

1. **Minimize Cold Starts**:
   - Use a uptime monitor (e.g., UptimeRobot) to ping your backend every 14 minutes
   - Free service: https://uptimerobot.com

2. **Optimize Database**:
   - Use connection pooling
   - Add indexes for frequently queried fields
   - Clean up old audit records periodically

3. **Frontend Optimization**:
   - Use SWR caching effectively
   - Implement loading states for cold starts
   - Add retry logic for failed requests
